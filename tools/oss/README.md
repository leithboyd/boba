# OOS harness — multi-block out-of-sample validation

Re-measures each feature's marginal value across **all 58 data blocks** (not just block 0), to tell which edges **persist out-of-sample** vs which were single-block luck. Built from the validated single-block feature notebooks (`notebooks/features/*.ipynb`, generated by `notebooks/features/builders/build_*.py`).

## Layout
- `oss_core.py` — per-block scaffold, lifted verbatim from the template builder: `load_block_arrays(idx)`, `build_grid(arrays)` (anchors, byb 100 ms price/rate targets, σ_ev/λ_ev, rate/vol controls), `marginal_ic(feat, target, controls)` (purged + embargoed walk-forward rank-IC over controls), `ic(feat, target)`, and a per-block npz cache (`load_cached(idx)`).
- `oss_features/<name>.py` — one module per feature, each exposing `NAME`, `HEAD` (`"price"`→scored vs `grid.price_target`, `"rate"`→`grid.rate_target`), and `compute(arrays, grid, spans=None) -> {leg: signed_array_on_grid}`. See `INTERFACE.md` for the exact contract. `flow_imbalance.py` is the reference implementation.
- `run_oss_all.py` — the sweep: fixes each feature's span to its **block-0 in-sample pick**, then scores all 58 blocks sequentially (one in memory at a time), checkpointing the per-block marginal-IC matrix to `oss_ic_matrix.csv`.
- `make_report.py` — turns the matrix into `../../oss_report.md` (per feature: block-0 IC, OOS mean±std over held-out blocks, %same-sign, PERSISTS/FADES verdict).
- `validate.py` — block-0 validation + bit-exact EMA oracle.
- `fixed_spans_block0.json` — the frozen block-0 span picks. `oss_ic_matrix.csv` — the latest result (58 blocks × feature joints + legs).

## Run
```bash
cd tools/oss
pixi run python run_oss_all.py     # ~58 heavy block builds, sequential, RAM-bounded; checkpoints per block
pixi run python make_report.py     # writes ../../oss_report.md
```
**Method (true OOS):** spans are fixed on block 0 only and reused unchanged on blocks 1..57, so the 1..57 estimate never saw the selection. block 0 = selection block; the honest number is the held-out mean.

## Cache
`load_cached(idx)` caches each block's arrays+grid as compressed npz (~236 MB/block). Default dir is `tools/oss/cache/` (gitignored, regenerable). Override with `OSS_CACHE_DIR=/path` to reuse an existing cache:
```bash
OSS_CACHE_DIR=/tmp/oss_harness/cache pixi run python run_oss_all.py
```

## Adding a feature
Copy `oss_features/flow_imbalance.py`, match `INTERFACE.md` (signed, length `len(grid.anchor_ts)`, strictly causal — build on `grid.merged_ts`, read at `grid.tick_at_anchor`), and validate on block 0 that `marginal_ic` reproduces the notebook's number before trusting the OOS run.

## Current result
12 of 15 features persist OOS; the cross-exchange OFI lead (`ofi_normalised`) and `microprice` lead the set. Full table in `../../summary.md` and `../../oss_report.md`. Caveat: the sweep validates the **marginal-IC** gate, not the echo-netted/feed-control gates — for echo-heavy features the tradeable net-of-echo number is the binding one.
